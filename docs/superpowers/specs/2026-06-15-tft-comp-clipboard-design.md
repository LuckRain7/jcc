# 金铲铲阵容码收藏工具 — 设计文档

日期：2026-06-15

## 1. 目标与背景

一个个人用的网页工具，用来收藏「金铲铲」阵容码。在电脑 web 端和手机浏览器端都能完整管理（增 / 删 / 改 / 查）阵容，数据存云端、跨设备同步。核心使用场景：在手机浏览器里点击某个阵容，**一键把阵容码复制到剪贴板**，再粘贴到游戏里。

### 范围约束
- **单用户**：只有作者本人使用，不做注册 / 多用户隔离，用一个简单密码门保护。
- **跨设备同步**：电脑录入的阵容手机能看到，反之亦然。
- **完整 CRUD，两端通用**：电脑和手机都能新增 / 编辑 / 删除。
- 字段精简：阵容名称、备注、阵容码。

### 非目标（YAGNI）
- 不做多用户 / 注册登录系统。
- 不做图片、标签、分类、版本号等额外字段（先不做）。
- 不做阵容码格式校验或与游戏对接。

## 2. 技术栈

- **前端 + 后端**：Next.js（App Router），用 Route Handlers 作为后端 API。
- **部署**：Vercel（自动 HTTPS —— 剪贴板 API 需要 HTTPS，正好满足）。
- **数据库**：Supabase（托管 Postgres）。

## 3. 架构与认证

### 架构
- 浏览器 **不直连 Supabase**。所有数据访问经过 Next.js 的 Route Handlers 中转。
- Supabase 的 **service role key 只放在服务端环境变量**，永不下发到浏览器。
- 因此不需要在 Supabase 配置 RLS（行级安全），由 API 层统一鉴权。

### 认证（私密访问）
- 单用户密码门，不做注册：
  1. 密码存在 Vercel 环境变量（如 `APP_PASSWORD`）。
  2. 访问站点先到登录页，输对密码 → 服务端下发一个**签名的 httpOnly cookie**（有效期 30 天）。
  3. 所有 API 路由校验该 cookie，无效则返回 401。
  4. Next.js middleware 拦截未登录对页面的访问，重定向到登录页。
- 手机和电脑各自登录一次，30 天内免输密码，共享同一云库。

### 关键环境变量
- `APP_PASSWORD` —— 访问密码。
- `SESSION_SECRET` —— 用于给 cookie 签名。
- `SUPABASE_URL`、`SUPABASE_SERVICE_ROLE_KEY` —— 服务端连 Supabase。

## 4. 数据模型

表 `compositions`：

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | uuid（主键，默认 `gen_random_uuid()`） | 自动生成 |
| `name` | text，非空 | 阵容名称 |
| `note` | text，可空 | 备注 / 说明 |
| `code` | text，非空 | 阵容码 |
| `created_at` | timestamptz，默认 `now()` | 创建时间 |
| `updated_at` | timestamptz，默认 `now()` | 更新时间，更新时刷新 |

- 列表默认按 `updated_at` 倒序（最近修改的排最前）。

## 5. API 设计

所有路由（除登录外）都校验密码 cookie，无效返回 401。

| 方法 | 路径 | 说明 |
|------|------|------|
| `POST` | `/api/login` | 校验密码，成功下发 httpOnly cookie |
| `POST` | `/api/logout` | 清除 cookie（可选） |
| `GET` | `/api/compositions` | 列出全部阵容（按 `updated_at` 倒序） |
| `POST` | `/api/compositions` | 新增（body: name, code, note） |
| `PUT` | `/api/compositions/:id` | 编辑，刷新 `updated_at` |
| `DELETE` | `/api/compositions/:id` | 删除 |

- 入参校验：`name`、`code` 必填；非法返回 400。

## 6. 界面与交互（移动优先）

### 登录页
- 一个密码输入框 + 登录按钮；密码错误给出提示。

### 列表页（主页）
- 移动优先的单列卡片布局，桌面端自动加宽 / 居中。
- 每张卡片：**阵容名称**（大字）、**备注**（小字，过长截断）。
- 卡片上有显眼的 **「复制阵容码」** 按钮：
  - 点击调用 `navigator.clipboard.writeText(code)`，成功弹出短暂 toast「已复制 ✓」。
  - **降级方案**：剪贴板 API 不可用 / 失败时，弹出一个含阵容码的文本框，提示用户长按选中复制。
- 顶部 **「+ 新增阵容」** 按钮。
- 每张卡片有 **编辑 / 删除**；删除需二次确认。

### 新增 / 编辑
- 简单表单：名称、阵容码、备注（多行）。以底部抽屉或独立页面呈现。

### 视觉风格
- 干净、紧凑、适合手机单手操作；点击目标足够大。成品级设计质感，避免默认简陋样式。

## 7. 错误处理
- API：鉴权失败 401、参数非法 400、Supabase 异常 500，前端统一提示。
- 剪贴板失败 → 走文本框降级方案。
- 网络请求失败 → toast 提示并允许重试。

## 8. 测试
- API 路由：鉴权（无 cookie / 错密码）、CRUD 正常路径、参数校验。
- 前端：复制交互（成功 + 降级）、列表渲染、表单增改删。
- 移动端真机 / 浏览器 devtools 验证剪贴板与布局。
